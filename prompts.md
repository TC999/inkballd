4. **空指针解引用**：原程序中依赖动态加载的资源（如模型、纹理、配置文件）在当前环境下未找到，返回 NULL 后直接调用。

## Core Principles (必须遵守)
1. **忠于原始逻辑**：反编译代码中的奇怪写法（如 `*(_DWORD *)(v1 + 4) = v2`、大量的强制类型转换、thiscall 转换等）通常有其底层内存布局原因。修复时以**消除编译错误**为第一优先级，**不要过度重构或主观臆断优化**，除非你确信那是反编译器的冗余。
2. **MSVC 兼容性**：所有修复必须符合 MSVC (VS2022) 的 C++ 编译器特性，注意 MSVC 对模板、绑定成员函数指针、匿名结构体/联合体以及内存对齐的特有要求。
3. **最小改动原则**：只修复导致编译失败的代码，不要顺手修改已经能正常编译的周边代码。
4. **缺失类型的处理**：如果遇到未定义的结构体、类或函数声明，请使用前向声明或提供最小的占位定义，并在注释中用 `[TODO]` 标记。
5. **警惕隐式依赖**：解除 `#if 0` 可能会暴露出之前未包含的头文件依赖，请敏锐地识别并补充。
